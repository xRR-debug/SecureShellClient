# SecureShell (Go + TLS Proxy)

## Основные возможности
Интерактивный клиент-сервер на Go для удаленного управления Shell. Весь трафик между клиентом и сервером проходит через защищенный **TLS-туннель**, что предотвращает перехват логинов, паролей и команд.

*   **🔒 TLS + Локальный прокси:** Полное шифрование данных между Windows и Linux. Локальный прокси на `127.0.0.1:1337`.
*   **🔑 Базовая авторизация:** Встроенная проверка логина и пароля на стороне сервера перед предоставлением доступа к Shell.

## Требования к системе
1.  **Go:** версия 1.18 или выше.
2.  **Windows:** Включенный компонент "Клиент Telnet".
3.  **Linux:** Установленные пакеты `xterm` и `telnet`.

## 💻 Совместимость 

| Компонент | ОС | Описание |
|:---:|:---:|:---|
| **Сервер** | 🐧 **Linux** | Требуется поддержка **PTY** (псевдо-терминалов) и оболочка `bash`. |
| **Клиент** | 🪟 **Windows** | Оптимизирован для работы со стандартным `telnet.exe` и кодировкой UTF-8. |

## 🔐 Настройка безопасности (TLS)

Для работы сервера необходимо сгенерировать SSL-сертификаты. Программа использует самоподписанный сертификат для шифрования трафика между клиентом и сервером.

### Генерация сертификатов
Выполните следующую команду в терминале (требуется установленный `openssl`):

```bash
openssl req -x509 -newkey rsa:4096 -keyout key.pem -out cert.pem -days 365 -nodes
```

Полученные файлы key.pem и cert.pem, положить рядом с client и server.

### Сборка Сервера на Linux
```bash
go build -o server server.go
chmod +x server
./server
```

### Сборка Клиента на Windows
```bash
# Стандартная сборка с консолью логов
go build -o client.exe client.go

# Скрытая сборка (только окно терминала)
go build -ldflags="-H windowsgui" -o client_silent.exe client.go
```
